# Task 0: Define User Stories and Mockups

## Introduction

This section defines the main user stories for the Subscriptions Organiser Platform and presents the initial wireframe mockups for the key screens of the MVP. The purpose of this task is to clarify the core system functionality from the user’s perspective and provide a visual representation of the primary interface screens.

The user stories are prioritised using the MoSCoW method to distinguish essential features from secondary and future features.

---

## Prioritized User Stories Using MoSCoW

### Must Have

1. As a new user, I want to create an account, so that I can securely access the platform.
2. As a registered user, I want to log in to my account, so that I can manage my subscriptions securely.
3. As a user, I want to add a new subscription, so that I can track its renewal date, cost, and category.
4. As a user, I want to view all my subscriptions in one place, so that I can manage them easily.
5. As a user, I want to edit my subscription details, so that I can keep my records up to date.
6. As a user, I want to delete subscriptions I no longer use, so that I can maintain an accurate list.

### Should Have

1. As a user, I want to see a monthly spending summary, so that I can better understand my recurring expenses.
2. As a user, I want to receive renewal reminders, so that I do not miss subscription payments.
3. As a user, I want to categorise my subscriptions, so that I can organise them by type.

### Could Have

1. As a user, I want to search my subscriptions, so that I can find them more quickly.
2. As a user, I want to filter subscriptions by status or category, so that I can organise them more efficiently.

### Won’t Have

1. As a user, I want AI-based subscription recommendations, so that I can receive smart suggestions.
2. As a user, I want bank integration, so that payments can be tracked automatically.
3. As a user, I want a mobile application version, so that I can manage subscriptions from my phone.

## Wireframe Descriptions

### Figure 1. Login / Register

This wireframe represents the authentication screen. It allows users to log in to their accounts or navigate to registration. The screen includes the essential fields for email and password, as well as a login button and a registration link.

### Figure 2. Dashboard

This wireframe represents the main dashboard of the platform. It provides a summary of total subscriptions, monthly spending, and the next renewal date. It also includes a recent subscriptions section to give users a quick overview of their active subscriptions.

### Figure 3. My Subscriptions

This wireframe shows the subscription management page. It includes a search bar, filter option, and a structured table displaying subscription details such as name, category, price, renewal date, status, and available actions like view, edit, and delete.

### Figure 4. Add Subscription

This wireframe represents the page where users can add a new subscription. It includes fields for subscription name, category, price, billing cycle, renewal date, reminder settings, and optional notes, as well as save and cancel buttons.
