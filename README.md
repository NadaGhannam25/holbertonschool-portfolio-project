# Holbertonschool Portfolio Project

This portfolio project is part of the Holberton School program. The goal of the project is to design and build a complete web application starting from the idea stage, through planning, development, and implementation. It allows students to apply the software engineering skills they have learned during the program while working collaboratively as a team.

---

## Stage 1: Team Formation and Idea Development

### Objective

The objective of this stage was to build a **collaborative and well-organized team**, explore multiple project ideas, and analyze them in order to identify the most suitable **Minimum Viable Product (MVP)** concept that aligns with the team’s interests, technical skills, and project goals.

---

### What We Did

During this stage, the team carried out several key activities:

- Conducted an initial team meeting to introduce all members and discuss their **backgrounds, technical skills, and interests**.
- Assigned **initial roles** to help organize the work and coordinate the activities during this stage.
- Established **Discord** as the primary communication platform to facilitate discussions and collaboration within the team.
- Performed **brainstorming sessions** where several project ideas were proposed and discussed.
- Evaluated each idea based on criteria such as:
  - Feasibility  
  - Potential user value  
  - Innovation  
  - Alignment with the team’s technical capabilities  

---

### Outcome

After discussing and evaluating the proposed ideas, the team selected the **Subscriptions Organiser Platform** as the MVP concept.

The platform aims to help users **organise their subscriptions in one place** while providing reminders for upcoming deadlines.

This idea was chosen because it is:

- Practical and useful for users  
- Feasible to implement within the project timeline  
- Well aligned with the team’s current technical skills
